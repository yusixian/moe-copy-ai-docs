## 基础操作

### 提取网页内容

#### 方法一：使用扩展图标

1. 访问目标网页
2. 点击浏览器工具栏中的 MoeCopy AI 图标
3. 扩展会自动分析并提取页面内容
4. 在弹出窗口中查看提取结果

#### 方法二：使用悬浮按钮

1. 在支持的网页上，右下角会显示悬浮按钮
2. 点击悬浮按钮快速打开提取面板
3. 如需隐藏悬浮按钮，可在设置中关闭

### 查看提取结果

提取完成后，您可以在不同标签页查看：

- **内容标签页**：显示提取的文章正文
- **元数据标签页**：显示标题、作者、日期等信息
- **图片标签页**：展示所有提取的图片
- **AI 摘要标签页**：使用 AI 处理内容

### 复制和导出

#### 复制单个字段

每个字段旁边都有复制按钮，点击即可复制到剪贴板：

- 标题
- 作者
- 发布日期
- 正文内容
- 清理后的内容

#### 批量导出

点击"下载"按钮，选择导出格式：

- **JSON**：包含所有结构化数据
- **Markdown**：格式化的文档
- **纯文本**：仅文本内容

## 高级功能

### 自定义选择器

为特定网站配置专属的提取规则：

1. 打开选项页面（右键扩展图标 → 选项）
2. 找到"选择器设置"部分
3. 添加自定义选择器：

```css
/* 内容选择器示例 */
.custom-article-content

/* 标题选择器示例 */
h1.article-title

/* 作者选择器示例 */
span.author-name

/* 日期选择器示例 */
time.publish-date
```

4. 点击"测试选择器"验证效果
5. 保存设置

### 切换提取模式

根据网页类型选择最适合的模式：

1. **选择器模式**
   - 适用于：新闻网站、博客、文档站点
   - 优势：精确、快速、可定制
   - 配置：可添加自定义选择器

2. **阅读模式**
   - 适用于：复杂布局、动态内容
   - 优势：智能识别、自动清理
   - 配置：可调整识别阈值

3. **混合模式**
   - 适用于：未知网站类型
   - 优势：兼容性最佳
   - 工作流程：选择器 → 阅读模式

### AI 功能使用

#### 配置 AI

1. 在选项页面找到"AI 设置"
2. 填写 API 配置：
   - API Key（必需）
   - API URL（可选，默认 OpenAI）
   - 模型选择

#### 使用 AI 摘要

1. 切换到"AI 摘要"标签页
2. 输入提示词，例如：
   - "总结这篇文章的主要观点"
   - "提取文章中的关键信息"
   - "将内容翻译成英文"
3. 点击"生成"按钮
4. AI 会流式输出结果

#### 提示词模板

使用内置变量创建动态提示词：

```
请总结以下文章：
标题：{{title}}
作者：{{author}}
内容：{{content}}

要求：
1. 提取3-5个要点
2. 每个要点不超过50字
3. 保持客观中立
```

### 开发者工具

#### 选择器测试

1. 打开选项页面
2. 在"开发者设置"中启用调试面板
3. 输入 CSS 选择器
4. 查看匹配结果和提取内容

#### 日志调试

调整日志级别查看详细信息：

- **Debug**：所有调试信息
- **Info**：一般信息
- **Error**：仅错误信息
- **Silent**：关闭日志

## 使用技巧

### 提高提取准确性

1. **针对特定网站优化**
   - 为常用网站添加自定义选择器
   - 测试不同提取模式的效果

2. **处理动态内容**
   - 等待页面完全加载
   - 使用手动触发模式
   - 切换到阅读模式

3. **优化 AI 输出**
   - 使用具体明确的提示词
   - 利用模板变量
   - 尝试不同的模型

### 批量抓取技巧

<Badge>v0.2.0 新增</Badge>

使用侧边栏的批量抓取功能，可以一次性抓取多个链接中的文档：

1. **打开侧边栏**
   - 右键点击扩展图标 → "在侧边栏打开"
   - 或使用快捷键打开侧边栏

2. **选择包含链接的区域**
   - 点击"选择元素区域"按钮
   - 在页面上点击包含多个链接的容器元素（如目录、列表等）

3. **管理提取的链接**
   - 查看自动提取的所有链接
   - 勾选需要抓取的链接
   - 可手动添加、编辑或删除链接
   - 启用"仅同域名"过滤无关链接
   - **正则过滤**：输入正则表达式筛选链接
     - 选择匹配目标：URL / 文本 / 两者
     - 选择模式：排除匹配项 / 仅保留匹配项
     - 使用预设快速应用常用规则（如排除图片、仅保留文档等）

4. **配置分页抓取**（可选）
   - 如果页面有分页，点击"选择下一页按钮"
   - 在页面上点击"下一页"按钮
   - 系统会自动翻页收集所有链接

5. **选择抓取策略**
   - **Fetch API**：速度最快，适合静态页面
   - **后台标签页**：支持 JS 渲染和登录态
   - **当前标签页**：可监控整个抓取过程

6. **开始抓取并导出**
   - 点击"开始抓取"
   - 查看实时进度，可随时暂停/继续
   - 抓取完成后，导出为 ZIP 文件或单独复制

### 移动端使用

在 Kiwi Browser 上使用时：

1. **优化显示**
   - 扩展会自动适应移动端
   - 可在设置中调整字体大小

2. **手势操作**
   - 支持滑动查看不同标签
   - 长按复制文本

## 常见问题处理

### 提取失败

- 检查网页是否完全加载
- 尝试切换提取模式
- 查看控制台错误信息

### AI 功能不可用

- 确认 API Key 正确
- 检查网络连接
- 查看 API 余额

### 性能优化

- 关闭不需要的功能
- 降低日志级别
- 清理历史记录
