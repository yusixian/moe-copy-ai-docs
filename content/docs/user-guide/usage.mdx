## 基础操作

### 提取网页内容

**方法一：扩展图标**

1. 访问目标网页
2. 点击工具栏中的 MoeCopy AI 图标
3. 查看自动提取的结果

**方法二：侧边栏单页抓取** <Badge>v0.2.1</Badge>

1. 右键扩展图标 → "在侧边栏打开"
2. 切换到"单页抓取"标签页
3. 点击"抓取"提取当前页面内容
4. 可使用 AI 总结按钮快速生成摘要

### 查看提取结果

- **内容标签页**：文章正文
- **元数据标签页**：标题、作者、日期等
- **图片标签页**：提取的图片
- **AI 摘要标签页**：AI 处理内容

### 复制和导出

**复制单个字段**：点击字段旁的复制按钮

**批量导出**：点击"下载"按钮，选择格式（JSON / Markdown / 纯文本）

## 高级功能

### 自定义选择器

为特定网站配置专属提取规则：

1. 打开选项页面（右键扩展图标 → 选项）
2. 添加自定义选择器：

```css
/* 内容 */
.custom-article-content

/* 标题 */
h1.article-title

/* 作者 */
span.author-name

/* 日期 */
time.publish-date
```

3. 点击"测试选择器"验证
4. 保存设置

### 提取模式

- **选择器模式**：精确、快速、可定制，适合新闻博客文档站点
- **阅读模式**：智能识别、自动清理，适合复杂布局、动态内容
- **混合模式**：兼容性最佳，选择器优先

### AI 功能

**配置 AI**：选项 → AI 设置 → 填写 API Key、API URL、模型

**使用 AI 摘要**：

1. 切换到"AI 摘要"标签页
2. 输入提示词，如"总结这篇文章的主要观点"
3. 点击"生成"

**提示词模板**：

```
请总结以下文章：
标题：{{title}}
作者：{{author}}
内容：{{content}}

要求：
1. 提取3-5个要点
2. 每个要点不超过50字
3. 保持客观中立
```

### 开发者工具

- **选择器测试**：选项 → 开发者设置 → 启用调试面板
- **日志级别**：Debug / Info / Error / Silent

## 批量抓取

<Badge>v0.2.0 新增</Badge>

使用侧边栏批量抓取多个链接中的文档：

1. 右键扩展图标 → "在侧边栏打开"
2. 切换到"批量抓取"标签页
3. 点击"选择元素区域"，在页面上点击包含链接的容器（如目录、列表）
4. 管理链接：勾选需抓取的链接，可手动添加/编辑/删除，启用同域名过滤或正则过滤
5. （可选）分页抓取：点击"选择下一页按钮"，系统会自动翻页收集链接
6. 选择抓取策略：
   - Fetch API：速度快，适合静态页面
   - 后台标签页：支持 JS 渲染和登录态
   - 当前标签页：可监控过程
7. 开始抓取，查看实时进度
8. 导出为 ZIP 或单独复制

### 移动端使用

在 Kiwi Browser 上使用时，扩展会自动适配移动端，支持滑动切换标签和长按复制。

## 常见问题处理

- **提取失败**：检查网页加载状态，尝试切换模式，查看控制台错误
- **AI 不可用**：确认 API Key 正确，检查网络和余额
- **性能优化**：关闭不需要的功能，降低日志级别，清理历史记录

## 实际案例

**新闻摘要**：打开文章 → 提取内容 → AI 提示"用 3 句话总结这篇新闻" → 复制摘要

**学术资料整理**：提取标题、作者、日期 → 导出 Markdown → 整理参考文献

**产品对比**：分别提取各平台商品信息 → 导出 JSON → 表格对比
