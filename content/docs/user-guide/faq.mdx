# 常见问题

这里整理了使用 MoeCopy AI 时最常见的问题和解决方案。

## 安装问题

### 为什么在 Chrome 商店找不到扩展？

MoeCopy AI 已经发布到 Chrome 网上应用店。如果搜索不到，可能是因为：

1. **地区限制**：某些地区可能无法访问
2. **搜索关键词**：尝试搜索 "MoeCopy AI" 或 "moe copy"
3. **直接访问**：使用[直接链接](https://chromewebstore.google.com/detail/moe-copy-ai/dfmlcfckmfgabpgbaobgapdfmjiihnck)

### 开发者模式安装失败怎么办？

常见原因和解决方法：

1. **构建失败**

   ```bash
   # 清理并重新安装依赖
   rm -rf node_modules pnpm-lock.yaml
   pnpm install
   pnpm build
   ```

2. **Node.js 版本过低**

   - 确保 Node.js >= 16
   - 使用 `node -v` 检查版本

3. **路径错误**
   - 加载扩展时选择 `build/chrome-mv3-prod` 目录
   - 不是项目根目录

### 移动端无法安装？

目前 Chrome 移动版不支持扩展，但您可以：

1. 使用 **Kiwi Browser**（Android）
2. 按照桌面版相同步骤安装
3. 界面会自动适配移动端

## 使用问题

### 为什么某些网页无法提取内容？

可能的原因：

1. **网页尚未加载完成**

   - 等待页面完全加载
   - 尝试刷新页面

2. **动态加载内容**

   - 切换到"手动抓取"模式
   - 等待内容加载后再点击扩展

3. **受保护的页面**

   - 某些网站（如银行、支付页面）禁止扩展访问
   - 这是浏览器的安全限制

4. **特殊网页结构**
   - 尝试切换提取模式
   - 添加自定义选择器

### 提取的内容不完整或错误？

解决方法：

1. **切换提取模式**

   ```
   选择器模式 → 阅读模式 → 混合模式
   ```

2. **配置自定义选择器**

   - 打开选项页面
   - 添加网站特定的选择器
   - 使用测试工具验证

3. **检查网页结构**
   - 使用浏览器开发者工具
   - 查看实际的 HTML 结构

### 悬浮按钮不显示？

检查以下设置：

1. **扩展设置**

   - 确认"显示悬浮按钮"已开启
   - 某些特殊页面可能不支持

2. **页面兼容性**

   - 新标签页、扩展页面不显示
   - 某些全屏应用可能冲突

3. **刷新页面**
   - 设置更改后需要刷新

## AI 功能问题

### AI 功能无法使用？

排查步骤：

1. **检查 API 配置**

   ```
   选项 → AI 设置 → 检查 API Key
   ```

2. **验证 API Key**

   - 确保 Key 正确且有效
   - 检查是否有使用额度

3. **网络连接**
   - 确保能访问 API 端点
   - 检查代理设置

### AI 响应速度慢？

优化建议：

1. **选择合适的模型**

   - gpt-3.5-turbo：速度快，效果好
   - gpt-4：效果最佳，速度较慢

2. **优化提示词**

   - 简洁明确的指令
   - 避免过长的输入

3. **使用自定义 API**
   - 配置更快的 API 端点
   - 使用本地部署的模型

### API 调用失败？

常见错误：

1. **401 Unauthorized**

   - API Key 无效或过期
   - 重新获取并配置

2. **429 Rate Limit**

   - 请求频率过高
   - 稍后重试

3. **网络错误**
   - 检查网络连接
   - 尝试更换 API 端点

## 性能问题

### 扩展运行缓慢？

优化方法：

1. **减少日志输出**

   ```
   选项 → 开发者设置 → 日志级别 → Error/Silent
   ```

2. **关闭不需要的功能**

   - 关闭自动抓取
   - 隐藏调试面板

3. **清理历史数据**
   - 定期清理 AI 聊天历史
   - 限制历史记录数量

### 页面加载变慢？

可能原因：

1. **自动抓取影响**

   - 切换到手动抓取模式
   - 仅在需要时使用

2. **复杂选择器**
   - 简化自定义选择器
   - 避免使用通配符

## 数据问题

### 如何备份数据？

1. **导出配置**

   - 选项页面 → 导出配置
   - 保存 JSON 文件

2. **Chrome 同步**

   - 登录 Google 账号
   - 自动同步设置

3. **手动备份**
   - 导出 AI 聊天历史
   - 保存自定义选择器

### 数据丢失怎么办？

预防和恢复：

1. **定期备份**

   - 每周导出一次配置
   - 使用云存储保存

2. **Chrome 同步恢复**

   - 重新登录账号
   - 等待同步完成

3. **从备份恢复**
   - 导入之前的配置文件
   - 重新配置 API

## 兼容性问题

### 不支持的浏览器？

支持情况：

| 浏览器  | 支持情况    | 说明               |
| ------- | ----------- | ------------------ |
| Chrome  | ✅ 完全支持 | 88+ 版本           |
| Edge    | ✅ 完全支持 | 基于 Chromium      |
| Firefox | ❌ 不支持   | 使用不同的扩展 API |
| Safari  | ❌ 不支持   | 不兼容             |
| Kiwi    | ✅ 支持     | Android 专用       |

### 与其他扩展冲突？

解决冲突：

1. **广告拦截器**

   - 将 MoeCopy AI 加入白名单
   - 或暂时禁用

2. **隐私保护扩展**

   - 可能阻止内容提取
   - 调整权限设置

3. **开发者工具**
   - 一般不冲突
   - 可以配合使用

## 隐私和安全

### MoeCopy AI 会收集哪些数据？

**不会收集**：

- 浏览历史
- 个人信息
- 网页内容

**本地存储**：

- 用户设置
- AI 聊天历史
- 自定义配置

### API Key 安全吗？

安全措施：

1. **本地加密存储**

   - 不会上传到服务器
   - 仅用于 AI 调用

2. **建议做法**
   - 使用专用 API Key
   - 定期更换
   - 设置使用限额

### 支持哪些网站类型？

MoeCopy AI 支持大部分网站类型：

**完全支持**：

- 新闻网站（如知乎、微博）
- 电商网站（如淘宝、京东）
- 博客和个人网站
- 论坛和社区

**部分支持**：

- SPA 单页应用（需手动抓取模式）
- 动态加载内容（需等待加载完成）

**不支持**：

- Chrome 内置页面（如 chrome://）
- 扩展程序页面
- 受保护页面（如网银）

## 快速故障排查

遇到问题时的检查清单：

1. ✓ 扩展是否最新版本
2. ✓ 页面是否完全加载
3. ✓ 控制台是否有错误
4. ✓ 设置是否正确保存
5. ✓ 网络连接是否正常

如果问题仍未解决，请提供：

- 浏览器版本
- 扩展版本
- 问题网页 URL
- 错误截图或日志
